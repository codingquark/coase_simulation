# Coase Theorem & AI Job Displacement Simulation

An agent-based economic simulation exploring the **Coase Theorem** applied to AI-driven job displacement. Workers hold the "right to employment"; firms must compensate them to automate. An interactive Streamlit dashboard lets you manipulate transaction costs, bargaining power, and market structure to see whether the Coasian prediction holds.


The dashboard opens in your browser. Select a **Preset Scenario** from the sidebar (or tune individual sliders), then click **Run Simulation**.

## Why This Exists

AI automation is accelerating. Policy debates often assume that who holds the rights (workers vs. firms) determines the outcome. The Coase Theorem challenges this: if transaction costs are low enough, the outcome is efficient *regardless* of who holds the rights — only the *distribution* of gains changes.

This simulation lets you test that claim directly:

- **Zero Transaction Costs** → Total surplus is maximized and invariant to bargaining power β
- **Positive Transaction Costs** → Efficient deals get blocked, deadweight loss appears, and β *does* affect total output

## The Economic Model

### Agents

| Agent | Default Count | Key Attributes |
|-------|:---:|---|
| **Workers** | 1,000 | Sector, skill (0–1), wage, employment status, retraining timer, bargaining power β |
| **Firms** | 50 | Sector, workforce, automation level, AI capability |
| **Sectors** | 5 | Manufacturing, Services, Tech, Healthcare, Creative |

### Sectors

Each sector has different susceptibility to AI automation:

| Sector | AI Applicability | Base Wage | Base Productivity |
|--------|:---:|:---:|:---:|
| Tech | 0.8 | 1.4 | 1.3 |
| Manufacturing | 0.7 | 1.0 | 1.0 |
| Services | 0.5 | 0.8 | 0.9 |
| Healthcare | 0.4 | 1.2 | 1.1 |
| Creative | 0.3 | 0.9 | 0.85 |

### Core Mechanism — Coasian Bargaining

For each worker position, the firm evaluates:

1. **Gross Surplus**

   ```
   S = (AI_capability × sector_applicability − worker_skill × wage) × horizon
   ```

2. **Transaction Cost**

   ```
   TC = base_TC × (1 + info_asymmetry + legal_friction + coordination_cost × n_workers_in_firm)
   ```

3. **Nash Bargaining Solution** (if S > TC)

   ```
   Worker compensation = β × (S − TC)
   Firm gain           = (1 − β) × (S − TC)
   ```

4. **Decision**
   - S ≤ 0 → No automation (AI not productive enough)
   - 0 < S ≤ TC → **Blocked** — efficient deal killed by transaction costs (deadweight loss = S)
   - S > TC → **Automate** — worker displaced with compensation, firm automation level rises

### AI Capability Growth

A logistic S-curve models AI progress:

```
ai_cap(t) = max_cap / (1 + exp(−growth_rate × (t − inflection)))
```

Three phases emerge: slow early progress → rapid transition → plateau.

### Labor Market

Unemployed workers match to vacancies via a **Cobb-Douglas matching function**:

```
Matches = efficiency × Unemployed^0.5 × Vacancies^0.5
```

Wages are set at marginal product adjusted for monopsony power:

```
Wage = productivity × (0.5 + skill) × labor_share × (1 − monopsony_markdown) × base_wage
```

New jobs are created through automation-driven innovation:

```
New jobs = base_creation + multiplier × automation_level × innovation_rate × 100
```

### Simulation Loop (5 Phases per Period)

| Phase | Action |
|:---:|---|
| 1 | **AI Growth** — Global AI capability advances along the S-curve |
| 2 | **Automation Decisions** — Each firm evaluates Coasian bargaining for every worker |
| 3 | **Labor Market Clearing** — Unemployed workers matched to vacancies, wages adjusted |
| 4 | **Job Creation** — New positions generated by automation-driven innovation |
| 5 | **State Updates** — Retraining timers tick down; completing workers gain +0.05 skill |

## Dashboard

### Sidebar Controls

All parameters are adjustable via sliders, grouped into collapsible sections. Every slider has a tooltip (hover the `?` icon) explaining its economic meaning.

**Preset Scenarios** load tuned parameter sets instantly:

| Preset | What It Demonstrates |
|--------|---|
| **Custom** | All defaults — your starting point |
| **Zero Transaction Costs** | Coase ideal: TC=0, no frictions. Surplus invariant to β |
| **High Friction** | Coase breakdown: TC=0.3, high asymmetry & legal costs. Deadweight loss appears |
| **Strong Worker Rights** | β=0.8, low TC. Workers capture most surplus |
| **Rapid AI Growth** | Fast S-curve (inflection at period 30). Sharp automation wave |

### Tabs

| Tab | Contents |
|-----|----------|
| **Overview** | KPI cards (employment rate, mean wage, displaced workers, blocked transactions) + time-series charts for employment, wages, AI capability, displacement vs. job creation |
| **Coase Analysis** | Surplus allocation (stacked area: worker compensation / firm gain / deadweight loss), blocked transactions over time, total surplus vs. comparison scenario |
| **Sector Dynamics** | Per-sector employment, automation levels, wages, and retraining pool size |
| **Distribution** | Wage histogram and skill-vs-wage scatter plot by sector, with a period slider |
| **Theory & Guide** | Full documentation of the Coase Theorem, economic model, formulas, suggested experiments, and glossary |

### Scenario Comparison

1. Run a simulation
2. Enable **Compare Mode** in the sidebar
3. Name and **Save** the current run
4. Change parameters and run again
5. Select the saved scenario from the dropdown — it overlays as dashed lines on all charts

This is the key workflow for demonstrating the Coase Theorem: compare TC=0 vs. TC=0.3 to see deadweight loss appear, or vary β with TC=0 to see that only the surplus *split* changes.

## Suggested Experiments

### 1. The Coase Theorem Test
Run **Zero Transaction Costs** with β=0.2, save it, then run with β=0.8 and compare. Total surplus curves should overlap — only the worker/firm split changes.

### 2. Transaction Costs Break Efficiency
Compare **Zero Transaction Costs** against **High Friction**. Total surplus drops; blocked transactions and deadweight loss appear.

### 3. Automation Wave Dynamics
Compare **Rapid AI Growth** (inflection at 30) with defaults (inflection at 50). Faster AI creates a sharper employment dip that recovers to the same long-run equilibrium.

### 4. Creative Destruction vs. Permanent Displacement
Run with default `job_creation_multiplier=0.1`, then set it to `0.0`. Without new job creation, employment never recovers.

### 5. Monopsony Power
Compare `monopsony_markdown=0.0` against `0.4`. Wages drop significantly under monopsony, even with the same productivity.
